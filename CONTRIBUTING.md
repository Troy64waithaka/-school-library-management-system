# Contributing to School Library Management System

Thank you for your interest in contributing! Here are some guidelines:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/school-library-management-system.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Start development: `npm run dev`

## Development Guidelines

- Follow the existing code style
- Write clear commit messages
- Test your changes before submitting
- Update documentation as needed

## Commit Message Format

```
[type]: Brief description

Optional detailed description explaining the change.

- Bullet point 1
- Bullet point 2
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

## Pull Request Process

1. Update README.md with any new features
2. Ensure all tests pass: `npm run lint`
3. Update CHANGELOG.md
4. Submit your PR with a clear description

## Code Style

- Use 2 spaces for indentation
- Use single quotes for strings
- Maximum line length: 100 characters
- Use meaningful variable names

## Questions?

Create an issue or reach out to the maintainers.

Thank you for contributing! 🎉
